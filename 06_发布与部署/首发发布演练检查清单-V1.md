# 首发发布演练检查清单 V1

> 目标：验证“新环境 1 小时内可起服务”，并确认首发主链路可用。
> 适用范围：`ui/V0_SAAS` + `apps/api` 单实例部署（文件存储）。

## 0. 演练前准备（T-1 天）

- [ ] 代码版本冻结：记录 commit id、发布人、发布时间窗。
- [ ] 环境变量就绪：`JWT_SECRET`、`KIMI_API_KEY`、`CONFIG_INTEGRITY_ON_STARTUP`。
- [ ] 备份策略可用：`npm run ops:backup:config` 可执行。
- [ ] 最小门禁全绿：`build:web`、`build:api`、`test:modules`、`test:rules`、`test:integration`。

## 1. 启动验证（T0）

- [ ] 执行 `docker compose up --build -d` 成功，无关键报错。
- [ ] API 健康检查：`/api/v1/health` 返回 200。
- [ ] Web 首页与登录页可访问。
- [ ] `logs/data-anomaly-repair.log` 可写入。

## 2. 主链路冒烟（T0+15min）

- [ ] 登录鉴权可用（正常登录、无 token 401、越权 403）。
- [ ] 版本机制主链路通过：`checkout` -> `checkin` -> `undo` -> `promote` -> `force-unlock`。
- [ ] 四大评估流程可保存与读取，且可关联版本。
- [ ] 团队协作可用：创建评审、更新状态、发表评论。
- [ ] WBS 可展示派生任务，字段完整（`moduleKey`、`linkedVersionCode` 等）。

## 3. 运维基线检查（T0+30min）

- [ ] CPU/内存/磁盘在可接受阈值内。
- [ ] 容器日志无持续性 error。
- [ ] 每日定时任务已配置：
  - [ ] 备份 `config/*`
  - [ ] 数据完整性校验
- [ ] 异常/修复日志留痕可追溯。

## 4. 回滚演练（T0+45min）

- [ ] 可回滚到上一个镜像/commit 并恢复服务。
- [ ] 可从最近一次 `config` 备份恢复数据。
- [ ] 回滚后健康检查与登录链路正常。

## 5. 结论记录（T0+60min）

- [ ] 记录演练结论：通过 / 有条件通过 / 不通过。
- [ ] 记录阻塞问题与责任人、修复截止时间。
- [ ] 输出“首发上线结论 + 二轮输入项”并归档到项目治理文档。
