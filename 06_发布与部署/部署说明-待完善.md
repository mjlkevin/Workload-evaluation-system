# 部署说明（待完善）

## 当前状态

仓库内 **尚未** 提交可用的 `Dockerfile` / `docker-compose.yml`。本地开发以 `npm run dev:api` 与 `npm run dev:web` 为主。

## 建议的最小生产形态（规划）

1. **API**：Node 进程运行 `apps/api` 构建产物，暴露 `PORT`（默认 3000）。
2. **Web**：`apps/web` 经 `npm run build:web` 产出静态资源，由 Nginx 托管；`location /api` 反代至 API。
3. **持久化目录**（需挂载卷）：
   - `config/templates`、`config/rules`
   - `config/auth`、`config/versions`
   - `exports/` 或配置的导出目录
4. **环境变量**：见根目录 `.env.example` 与 `docs/ENVIRONMENT.md`。
5. **TLS**：由 Nginx 或前置负载均衡终止 HTTPS。

## 首次部署检查清单（草案）

- [ ] `JWT_SECRET` 已更换为强随机值  
- [ ] `KIMI_API_KEY` 等密钥未写入镜像层  
- [ ] 导出目录磁盘空间与清理策略（TTL）  
- [ ] `config/*` 备份策略
