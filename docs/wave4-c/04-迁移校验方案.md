# Wave4-C · 迁移校验方案

> 目标：确保 v1 → v2 迁移结果正确、完整、可追溯
> 基线：7c00005

---

## 一、校验项列表

| # | 校验项 | 检查方式 | 校验失败后定位 | 回滚/补迁 | 执行顺序 |
|---|---|---|---|---|---|
| 1 | 数量对账 | `SELECT COUNT(*)` vs JSON 数组长度 | 对比差异数 | 补迁缺失记录 | 每批写入后立即 |
| 2 | 字段完整性 | 抽样检查关键字段非空 | 定位空字段记录 | 补迁或标记异常 | 每批写入后立即 |
| 3 | 主键唯一性 | `SELECT assessment_version_id, COUNT(*) GROUP BY HAVING COUNT(*) > 1` | 定位重复主键 | 删除重复后补迁 | 全量写入后 |
| 4 | 关联引用检查 | 检查 `owner_user_id` 是否存在于 users 表 | 定位孤儿引用 | 补迁对应 user 或标记 | 全量写入后 |
| 5 | payload 可解析性 | 抽样 `jsonb` 字段能否正常 JSON.parse | 定位损坏记录 | 重新转换后补迁 | 全量写入后 |
| 6 | 状态映射正确性 | 抽样核对 v1 status → v2 status | 定位映射错误 | 修正映射后重跑 | 全量写入后 |
| 7 | 重跑一致性 | 重跑迁移脚本，expect `skipped = 全部` | 若 `inserted > 0` 说明幂等失效 | 修复幂等逻辑 | 全量写入后 |
| 8 | 异常记录归档 | 收集 transform/write 失败的记录 | 查看 error log | 人工确认后补迁 | 全量写入后 |

## 二、校验实现

### 2.1 数量对账校验器

```typescript
export class CountValidator implements Validator {
  name = "count-reconciliation";

  async validate(migrator: BaseMigrator): Promise<ValidationResult> {
    const sourceCount = migrator.loadSource().length;
    const targetCount = await db
      .select({ count: sql<number>`count(*)` })
      .from(sql.raw(migrator.targetTable));

    const actualCount = targetCount[0]?.count || 0;
    const diff = sourceCount - actualCount;

    return {
      passed: diff === 0,
      message: diff === 0
        ? `数量一致: ${sourceCount}`
        : `数量不一致: 源=${sourceCount}, 目标=${actualCount}, 差=${diff}`,
      details: { sourceCount, targetCount: actualCount, diff },
    };
  }
}
```

### 2.2 字段完整性校验器

```typescript
export class FieldCompletenessValidator implements Validator {
  name = "field-completeness";
  requiredFields = ["assessment_version_id", "version_code", "status", "payload"];

  async validate(migrator: BaseMigrator): Promise<ValidationResult> {
    const nullChecks = this.requiredFields.map((field) =>
      db
        .select({ count: sql<number>`count(*)` })
        .from(sql.raw(migrator.targetTable))
        .where(sql.raw(`${field} IS NULL`))
    );

    const results = await Promise.all(nullChecks);
    const issues = results
      .map((r, i) => ({ field: this.requiredFields[i], nullCount: r[0]?.count || 0 }))
      .filter((x) => x.nullCount > 0);

    return {
      passed: issues.length === 0,
      message: issues.length === 0
        ? "所有必填字段均无空值"
        : `发现 ${issues.length} 个字段存在空值`,
      details: { issues },
    };
  }
}
```

### 2.3 主键唯一性校验器

```typescript
export class UniqueKeyValidator implements Validator {
  name = "unique-key";

  async validate(migrator: BaseMigrator): Promise<ValidationResult> {
    const duplicates = await db
      .select({ id: sql<string>`${sql.raw(migrator.targetTable)}.assessment_version_id` })
      .from(sql.raw(migrator.targetTable))
      .groupBy(sql.raw("assessment_version_id"))
      .having(sql`count(*) > 1`);

    return {
      passed: duplicates.length === 0,
      message: duplicates.length === 0
        ? "主键无重复"
        : `发现 ${duplicates.length} 个重复主键`,
      details: { duplicateIds: duplicates.map((d) => d.id) },
    };
  }
}
```

### 2.4 关联引用校验器

```typescript
export class ForeignReferenceValidator implements Validator {
  name = "foreign-reference";

  async validate(): Promise<ValidationResult> {
    const orphanRecords = await db
      .select({
        assessmentVersionId: assessmentVersions.assessmentVersionId,
        ownerUserId: assessmentVersions.ownerUserId,
      })
      .from(assessmentVersions)
      .leftJoin(users, eq(assessmentVersions.ownerUserId, users.userId))
      .where(and(
        sql`${assessmentVersions.ownerUserId} IS NOT NULL`,
        sql`${users.userId} IS NULL`
      ));

    return {
      passed: orphanRecords.length === 0,
      message: orphanRecords.length === 0
        ? "所有 owner_user_id 引用有效"
        : `发现 ${orphanRecords.length} 条孤儿引用`,
      details: { orphanRecords },
    };
  }
}
```

### 2.5 Payload 可解析性校验器

```typescript
export class PayloadParseableValidator implements Validator {
  name = "payload-parseable";
  sampleSize = 50;

  async validate(migrator: BaseMigrator): Promise<ValidationResult> {
    const samples = await db
      .select({ payload: assessmentVersions.payload })
      .from(assessmentVersions)
      .limit(this.sampleSize);

    const unparsable = samples.filter((s) => {
      try {
        JSON.parse(JSON.stringify(s.payload));
        return false;
      } catch {
        return true;
      }
    });

    return {
      passed: unparsable.length === 0,
      message: unparsable.length === 0
        ? `抽样 ${samples.length} 条 payload 均可解析`
        : `抽样中发现 ${unparsable.length} 条 payload 不可解析`,
      details: { sampleSize: samples.length, unparsableCount: unparsable.length },
    };
  }
}
```

### 2.6 状态映射正确性校验器

```typescript
export class StatusMappingValidator implements Validator {
  name = "status-mapping";
  expectedMappings: Record<string, string> = {
    draft: "draft",
    reviewed: "checked_in",
    published: "promoted",
    archived: "sealed",
  };

  async validate(): Promise<ValidationResult> {
    const samples = await db
      .select({
        id: assessmentVersions.assessmentVersionId,
        versionCode: assessmentVersions.versionCode,
        status: assessmentVersions.status,
        payload: assessmentVersions.payload,
      })
      .from(assessmentVersions)
      .limit(50);

    const mismatches = samples.filter((s) => {
      const v1Status = (s.payload as any)?._v1Meta?.status;
      if (!v1Status) return false;
      const expectedV2Status = this.expectedMappings[v1Status];
      return expectedV2Status && s.status !== expectedV2Status;
    });

    return {
      passed: mismatches.length === 0,
      message: mismatches.length === 0
        ? "抽样状态映射均正确"
        : `抽样中发现 ${mismatches.length} 条状态映射异常`,
      details: { mismatches: mismatches.map((m) => ({ id: m.id, expected: this.expectedMappings[(m.payload as any)._v1Meta.status], actual: m.status })) },
    };
  }
}
```

### 2.7 重跑一致性校验器

```typescript
export class RerunConsistencyValidator implements Validator {
  name = "rerun-consistency";

  async validate(migrator: BaseMigrator): Promise<ValidationResult> {
    const result = await migrator.run({ dryRun: false });

    const passed = result.inserted === 0 && result.errors.length === 0;
    return {
      passed,
      message: passed
        ? "重跑一致性通过：无新增插入，无错误"
        : `重跑一致性失败: 新增 ${result.inserted} 条, 错误 ${result.errors.length} 条`,
      details: result,
    };
  }
}
```

## 三、校验执行顺序

```
[每批写入后]
  ├──→ CountValidator（数量对账）
  └──→ FieldCompletenessValidator（字段完整性）

[全量写入后]
  ├──→ UniqueKeyValidator（主键唯一性）
  ├──→ ForeignReferenceValidator（关联引用）
  ├──→ PayloadParseableValidator（payload 可解析）
  ├──→ StatusMappingValidator（状态映射正确性）
  ├──→ RerunConsistencyValidator（重跑一致性）
  └──→ 输出最终校验报告
```

## 五、回滚策略

| 场景 | 回滚方式 | 说明 |
|---|---|---|
| 单条写入失败 | 无需回滚，跳过即可 | 幂等设计保证 |
| 批量写入部分失败 | 逐条回退或标记异常 | 记录错误，人工处理 |
| 校验发现严重数据错误 | 执行 Migrator.rollback() | 清理已写入数据，修复后重跑 |
| 重跑一致性失败 | 修复幂等逻辑后重跑 | 定位未命中 exists 检查的记录 |

**回滚命令**：

```bash
# 按 Migrator 回滚
npx tsx scripts/migrate-v1-to-v2/cli.ts --rollback users
npx tsx scripts/migrate-v1-to-v2/cli.ts --rollback version-records

# 全部回滚
npx tsx scripts/migrate-v1-to-v2/cli.ts --rollback-all
```
