# Wave4-C · v1 → v2 迁移映射报告

> 目标：将 v1 JSON 资产映射到 v2 PostgreSQL schema
> 基线：7c00005

---

## 一、映射总览

| v1 资产 | v2 目标表 | 映射复杂度 | 关键风险 |
|---|---|---|---|
| `config/auth/users.json` | 新建 `users` 表 | 低 | 密码哈希兼容、role 映射 |
| `config/auth/invite-codes.json` | 新建 `invite_codes` 表 | 低 | 量小，简单 |
| `config/versions/records.json` | `assessment_versions` | 中 | status 映射、payload 兼容 |
| `config/versions/version-code-rules.json` | 新建 `version_code_rules` 表 | 低 | 结构简单 |
| `config/templates/example-template.json` | 新建 `templates` + `template_items` | 高 | 体积大、关系复杂 |
| `config/teams/store.json` | 新建 `teams` + `team_members` + `team_reviews` | 高 | 乐观锁、audit log |
| `config/system/*.json` | 保持 JSON 或新建 `system_configs` | 低 | API Key 敏感 |

## 二、详细映射

### 2.1 users.json → users 表

**建议 schema（新建）**：

```typescript
export const users = pgTable("users", {
  userId: uuid("user_id").primaryKey(),
  username: text("username").notNull().unique(),
  passwordHash: text("password_hash").notNull(),
  role: text("role", { enum: ["admin", "sub_admin", "user"] }).notNull().default("user"),
  status: text("status", { enum: ["active", "disabled"] }).notNull().default("active"),
  createdAt: timestamp("created_at", { withTimezone: true }).notNull(),
  lastLoginAt: timestamp("last_login_at", { withTimezone: true }),
});
```

| v1 字段 | v2 字段 | 类型变化 | 转换逻辑 | 保留 |
|---|---|---|---|---|
| `id` | `user_id` | string → uuid | 直接复用原 UUID | ✅ |
| `username` | `username` | string → text | 直接复制 | ✅ |
| `passwordHash` | `password_hash` | string → text | 原样复制（bcrypt 兼容） | ✅ |
| `role` | `role` | string → enum | 直接映射：`admin`/`sub_admin`/`user` | ✅ |
| `status` | `status` | string → enum | `active`→`active`, `disabled`→`disabled` | ✅ |
| `createdAt` | `created_at` | ISO string → timestamp | `new Date(createdAt)` | ✅ |
| `lastLoginAt` | `last_login_at` | ISO string → timestamp | `new Date(lastLoginAt)` | ✅ |

**无需人工确认**。

### 2.5 templates/example-template.json → templates + template_items 表

**建议 schema（新建）**：

```typescript
export const templates = pgTable("templates", {
  templateId: text("template_id").primaryKey(),
  templateVersion: text("template_version"),
  templateName: text("template_name").notNull(),
  createdAt: timestamp("created_at", { withTimezone: true }).defaultNow().notNull(),
  updatedAt: timestamp("updated_at", { withTimezone: true }).defaultNow().notNull(),
});

export const templateItems = pgTable("template_items", {
  templateItemId: text("template_item_id").primaryKey(),
  templateId: text("template_id").notNull(),
  groupId: text("group_id").notNull(),
  groupName: text("group_name").notNull(),
  itemName: text("item_name").notNull(),
  standardDays: real("standard_days").notNull(),
  cloudProduct: text("cloud_product"),
  skuName: text("sku_name"),
});
```

**迁移复杂度：高**。
- 原 JSON 体积 414KB，含大量 groups 和 items
- 需拆分为主表 + 子表
- 前端 `getTemplateDetail()` 需改为 JOIN 查询

**建议**：P1 优先级，可延后到 Wave 4 后期或 Wave 5。

### 2.7 system/*.json → system_configs 表（或保持 JSON）

**建议**：保持 JSON 文件，不迁 PG。

原因：
1. 配置类数据变更频率低
2. `requirement-settings.json` 含敏感信息（Kimi API Key）
3. 当前 `system.routes.ts` 已提供完整 CRUD API
4. 迁入 PG 需额外加密字段，增加复杂度

**例外**：若未来需要多环境配置热切换，可新建 `system_configs(key, value, env)` 表。

## 四、风险总结

1. **Status 语义映射**：v1 `published` → v2 `promoted` 是否准确，需业务确认
2. **Payload 兼容性**：v1 records 的 payload 直接放入 v2 jsonb，前端读取方式不变，但字段路径可能不同
3. **并发控制**：teams 的乐观锁需重新设计
4. **敏感信息**：requirement-settings.json 中的 API Key 严禁明文入 PG
5. **模板体积**：414KB 模板数据拆分后可能导致大量 INSERT，需分批执行
