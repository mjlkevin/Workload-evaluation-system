# Wave4-C2 · 迁移报告增强方案

## 结论速览

迁移 CLI 已支持 4 种运行模式 + JSON 报告导出。`--dry-run` 确保零副作用，`--validate` / `--comprehensive-validate` 支持独立校验，`--report` 支持结构化报告输出供 CI/CD 集成。

---

## 1. 命令行为

### 1.1 模式总览

| 模式 | 命令 | 写库 | 校验 | 报告 |
|-----|------|------|------|------|
| **LIVE** | `npm run migration:v1-to-v2` | ✅ 写入 | ✅ 基础+深度 | 控制台 |
| **DRY-RUN** | `--dry-run` | ❌ 不写入 | ❌ 跳过 | 控制台+可选 JSON |
| **VALIDATE** | `--validate` | ❌ 不写入 | ✅ 仅基础数量 | 控制台+可选 JSON |
| **COMPREHENSIVE** | `--comprehensive-validate` | ❌ 不写入 | ✅ 全部 14 项 | 控制台+可选 JSON |

### 1.2 使用示例

```bash
# 预览模式（零副作用）
npm run migration:v1-to-v2 -- --dry-run

# 预览并导出报告
npm run migration:v1-to-v2 -- --dry-run --report ./dry-run-report.json

# 仅基础校验
npm run migration:v1-to-v2 -- --validate

# 深度校验（推荐用于 CI/CD gate）
npm run migration:v1-to-v2 -- --comprehensive-validate

# 深度校验 + 导出 JSON 报告
npm run migration:v1-to-v2 -- --comprehensive-validate --report ./validation-report.json

# 正常迁移（自动执行基础+深度校验）
npm run migration:v1-to-v2

# 正常迁移 + 导出完整报告
npm run migration:v1-to-v2 -- --report ./migration-report.json
```

## 2. 模式差异详解

### 2.1 LIVE 模式
- **行为**：执行全部 3 个 migrator，实际 INSERT 到 PG
- **校验**：迁移完成后自动执行基础校验 + 深度校验
- **适用**：生产环境首次迁移、修复后重跑
- **风险**：会修改数据库，但幂等（已存在记录自动跳过）

### 2.2 DRY-RUN 模式
- **行为**：遍历全部源数据，模拟 INSERT 但不执行 SQL
- **校验**：跳过（因为未实际写库，无法校验 PG 状态）
- **适用**：迁移前预览、验证源数据可读性、估算迁移耗时
- **输出**：逐条显示"将插入"记录及关键字段

### 2.3 VALIDATE 模式
- **行为**：不执行任何 migrator，仅对比源 JSON 与 PG 行数
- **校验**：3 项基础数量对账
- **适用**：快速确认迁移是否完整、定时巡检
- **速度**：最快（< 100ms）

### 2.4 COMPREHENSIVE-VALIDATE 模式
- **行为**：不执行任何 migrator，执行全部 14 项深度校验
- **校验**：数量、唯一性、完整性、payload、重复数据
- **适用**：CI/CD 质量门禁、迁移验收、定期审计
- **退出码**：失败项 > 0 时 exit 1，否则 exit 0

## 3. 日志输出

### 3.1 迁移阶段日志

```
--- 1/3 迁移 users ---
[migrate:users] 跳过已存在: mjlkevin
[migrate:users] 已插入: elly
源=4 插入=1 跳过=3 错误=0
```

### 3.2 校验阶段日志

```
--- 迁移后深度校验 ---
总计: 14 项 | ✅ 通过=13 | ❌ 失败=0 | ⚠️ 警告=1

[COUNT]
  ✅ users 数量对账: 源=4 / PG=4
  ...

[DUPLICATE]
  ⚠️ records.json 源内 versionCode 重复: 重复项: GL-UT-03(3次)
```

### 3.3 日志级别
- `INFO`：正常插入、跳过、校验通过
- `WARN`：源内重复、payload 字段缺失
- `ERROR`：插入失败、唯一性冲突、必填字段为空

## 5. 使用方式

### 5.1 本地开发
```bash
# 修改源数据后预览
npm run migration:v1-to-v2 -- --dry-run

# 确认无误后执行
npm run migration:v1-to-v2
```

### 5.2 CI/CD 集成
```bash
# 在部署流水线中加入质量门
npm run migration:v1-to-v2 -- --comprehensive-validate --report /tmp/mig-report.json

# 根据报告判断是否通过
if [ $(jq '.comprehensive.summary.failed' /tmp/mig-report.json) -gt 0 ]; then
  echo "迁移校验失败"
  exit 1
fi
```

### 5.3 定期审计
```bash
# 每日定时执行深度校验
0 2 * * * cd /app && npm run migration:v1-to-v2 -- --comprehensive-validate --report /var/log/migration-audit/$(date +%Y%m%d).json
```
