# Wave4-C · 迁移脚本方案

> 目标：v1 JSON → v2 PostgreSQL 可重复执行、可扩展的迁移脚本骨架
> 基线：7c00005

---

## 一、设计原则

1. **幂等性**：重复执行不报错、不重复插入
2. **可追溯**：每条迁移记录都有日志、有标识
3. **可扩展**：新增资产类型时只需注册新的 Migrator
4. **可回滚**：失败时可清理已写入数据
5. **可验证**：迁移后自动执行校验

---

**CLI 用法**：

```bash
# 全量迁移
npx tsx scripts/migrate-v1-to-v2/cli.ts --all

# 只迁移指定资产
npx tsx scripts/migrate-v1-to-v2/cli.ts --migrators users,version-records

# 干跑（不写入数据库）
npx tsx scripts/migrate-v1-to-v2/cli.ts --all --dry-run

# 强制重跑（清除已有数据后重新迁移）
npx tsx scripts/migrate-v1-to-v2/cli.ts --all --force

# 只执行校验
npx tsx scripts/migrate-v1-to-v2/cli.ts --validate-only
```

## 三、核心函数拆分

### 3.1 BaseMigrator 抽象类

```typescript
export abstract class BaseMigrator<TSource, TTarget> {
  abstract readonly name: string;
  abstract readonly sourcePath: string;
  abstract readonly targetTable: string;

  /** 读取 v1 JSON 源数据 */
  abstract loadSource(): TSource[];

  /** 单条转换 */
  abstract transform(source: TSource): TTarget | null;

  /** 写入 v2 PG */
  abstract write(targets: TTarget[]): Promise<{ inserted: number; skipped: number; errors: MigrationError[] }>;

  /** 幂等检查：该记录是否已存在 */
  abstract exists(source: TSource): Promise<boolean>;

  /** 清理已迁移数据（回滚用） */
  abstract rollback(): Promise<void>;

  /** 执行完整迁移流程 */
  async run(options: { dryRun?: boolean; force?: boolean } = {}): Promise<MigrationResult> {
    const sources = this.loadSource();
    const targets: TTarget[] = [];
    const errors: MigrationError[] = [];
    let skipped = 0;

    for (const source of sources) {
      if (!options.force && await this.exists(source)) {
        skipped++;
        continue;
      }
      const target = this.transform(source);
      if (target) targets.push(target);
      else errors.push({ source, reason: "transform_failed" });
    }

    if (!options.dryRun) {
      const writeResult = await this.write(targets);
      return { ...writeResult, skipped, errors: [...errors, ...writeResult.errors] };
    }

    return { inserted: targets.length, skipped, errors };
  }
}
```

### 3.2 具体 Migrator 示例（users）

```typescript
export class UsersMigrator extends BaseMigrator<V1User, UserInsert> {
  name = "users";
  sourcePath = "config/auth/users.json";
  targetTable = "users";

  loadSource(): V1User[] {
    const raw = JSON.parse(fs.readFileSync(this.sourcePath, "utf-8"));
    return raw.users || [];
  }

  transform(source: V1User): UserInsert {
    return {
      userId: source.id,
      username: source.username,
      passwordHash: source.passwordHash,
      role: source.role,
      status: source.status,
      createdAt: new Date(source.createdAt),
      lastLoginAt: source.lastLoginAt ? new Date(source.lastLoginAt) : null,
    };
  }

  async exists(source: V1User): Promise<boolean> {
    const [row] = await db.select().from(users).where(eq(users.userId, source.id));
    return !!row;
  }

  async write(targets: UserInsert[]): Promise<{ inserted: number; skipped: number; errors: MigrationError[] }> {
    if (targets.length === 0) return { inserted: 0, skipped: 0, errors: [] };
    const result = await db.insert(users).values(targets).onConflictDoNothing().returning();
    return { inserted: result.length, skipped: targets.length - result.length, errors: [] };
  }

  async rollback(): Promise<void> {
    await db.delete(users);
  }
}
```

### 3.3 复杂 Migrator 示例（version-records）

```typescript
export class VersionRecordsMigrator extends BaseMigrator<V1Record, AssessmentVersionInsert> {
  name = "version-records";
  sourcePath = "config/versions/records.json";
  targetTable = "assessment_versions";

  loadSource(): V1Record[] {
    const raw = JSON.parse(fs.readFileSync(this.sourcePath, "utf-8"));
    return raw.records || [];
  }

  transform(source: V1Record): AssessmentVersionInsert | null {
    // 状态映射
    const statusMap: Record<string, string> = {
      draft: "draft",
      reviewed: "checked_in",
      published: "promoted",
      archived: "sealed",
    };

    const v2Status = statusMap[source.status];
    if (!v2Status) {
      logger.warn(`Unknown status "${source.status}" for record ${source.id}, skipping`);
      return null;
    }

    // payload 转换：将 v1 特有字段合并进 payload
    const enrichedPayload = {
      ...source.payload,
      _v1Meta: {
        type: source.type,
        templateId: source.templateId,
        createdByUserId: source.createdByUserId,
        createdByUsername: source.createdByUsername,
        updatedByUserId: source.updatedByUserId,
        updatedByUsername: source.updatedByUsername,
        versionDocStatus: source.versionDocStatus,
        majorLetter: source.majorLetter,
        minorNumber: source.minorNumber,
        baseCode: source.baseCode,
        isHistoricalArchive: source.isHistoricalArchive,
        archivedAt: source.archivedAt,
        checkoutAt: source.checkoutAt,
      },
    };

    return {
      assessmentVersionId: source.id,
      versionCode: source.versionCode,
      revisionType: "initial",
      status: v2Status as any,
      payload: enrichedPayload,
      ownerUserId: source.ownerUserId,
      checkedOutByUserId: source.checkedOutByUserId || null,
      createdAt: new Date(source.createdAt),
      updatedAt: new Date(source.updatedAt),
    };
  }

  async exists(source: V1Record): Promise<boolean> {
    const [row] = await db
      .select()
      .from(assessmentVersions)
      .where(eq(assessmentVersions.assessmentVersionId, source.id));
    return !!row;
  }

  async write(targets: AssessmentVersionInsert[]): Promise<{ inserted: number; skipped: number; errors: MigrationError[] }> {
    // 分批写入，每批 100 条
    const batchSize = 100;
    let inserted = 0;
    let skipped = 0;
    const errors: MigrationError[] = [];

    for (let i = 0; i < targets.length; i += batchSize) {
      const batch = targets.slice(i, i + batchSize);
      try {
        const result = await db
          .insert(assessmentVersions)
          .values(batch)
          .onConflictDoNothing()
          .returning();
        inserted += result.length;
        skipped += batch.length - result.length;
      } catch (e) {
        errors.push({ batchIndex: i, reason: String(e) });
      }
    }

    return { inserted, skipped, errors };
  }

  async rollback(): Promise<void> {
    // 只回滚由本脚本写入的记录（通过 payload._v1Meta 标识）
    await db
      .delete(assessmentVersions)
      .where(sql`${assessmentVersions.payload}->'_v1Meta' IS NOT NULL`);
  }
}
```

## 五、幂等策略

1. **逐条检查**：每条记录写入前查询目标表，已存在则跳过
2. **ON CONFLICT DO NOTHING**：PG 层面兜底
3. **_v1Meta 标识**：payload 中嵌入 `_v1Meta` 标签，便于识别迁移来源
4. **force 模式**：`--force` 时先 DELETE 再 INSERT

---

## 六、错误处理

| 错误类型 | 处理方式 |
|---|---|
| JSON 解析失败 | 记录错误，跳过该文件，继续其他 Migrator |
| 单条 transform 失败 | 记录错误，跳过该条，继续其他记录 |
| 批量写入失败 | 降级为逐条写入，失败的逐条记录错误 |
| 未知 status 映射 | 记录 warn，跳过该条 |
| 必填字段缺失 | 记录 error，跳过该条 |

## 七、执行流程

```
[开始]
  │
  ▼
[加载配置] ──→ 读取 config.json + 命令行参数
  │
  ▼
[初始化 Logger]
  │
  ▼
[注册 Migrators]
  │
  ▼
[循环每个 Migrator]
  │
  ├──→ [loadSource] 读取 v1 JSON
  │
  ├──→ [transform] 逐条转换
  │
  ├──→ [exists 检查] 幂等判断
  │
  ├──→ [write] 写入 PG（dryRun 时跳过）
  │
  └──→ [记录结果]
  │
  ▼
[执行校验]
  │
  ▼
[输出报告]
  │
  ▼
[结束]
```
