# Wave4-C2 · 历史项目库增强建议

## 结论速览

当前 `history_projects` + `findSimilar` 已能支撑基础的"老客户继承"场景，但相似度算法较简单（行业 50 分 + 规模 30 分 + 模块重叠），且缺少 `projectName`、`风险等级`、`实际工期偏差原因`等前端展示必需字段。建议优先补 `projectName` 和增强相似度维度，再考虑为不同角色提供差异化摘要视图。

---

## 1. 当前能力评估

### 1.1 Schema 能力

| 字段 | 当前状态 | 评估 |
|-----|---------|------|
| `industry` / `scale` | ✅ 有，且带索引 | 搜索维度完整 |
| `modules` (jsonb) | ✅ 有 | 可存模块清单，但无 schema 约束 |
| `estimatedDays` / `actualDays` | ✅ 有 | 可计算偏差 |
| `estimatedCost` / `actualCost` | ✅ 有 | 可计算偏差 |
| `riskTags` (jsonb) | ✅ 有 | 可存风险标签 |
| `delayReason` | ✅ 有 | 可存延期原因 |
| `projectName` / `title` | ❌ 无 | **列表视图无法展示可读名称** |
| `customerName` | ❌ 无 | 无法识别老客户 |
| `projectStatus` | ❌ 无 | 无法区分已交付/已取消 |
| `teamSize` | ❌ 无 | 资源成本参考维度缺失 |
| `completionRate` | ❌ 无 | 需由前端计算 actual/estimated |

### 1.2 findSimilar 算法

当前算法：
- 行业完全匹配：+50 分
- 行业部分匹配：+30 分
- 规模关键词重叠：+20 分（每个关键词）
- 模块重叠：+10 分（每个模块）
- 返回 top 5

**评估**：基础可用，但维度单一，无权重调参，无置信度输出。

## 3. 相似度规则增强建议

### 3.1 当前问题
1. 分数无上限，不同项目之间分数不可比
2. 无权重配置，行业匹配永远比模块匹配重要
3. 无负向评分，规模差异大时不会降权
4. 无归一化，无法输出"相似度 85%"这样的直观指标

### 3.2 建议增强

#### 方案 A：归一化加权评分（推荐）

```
总分 = 行业分 * 0.4 + 规模分 * 0.2 + 模块分 * 0.3 + 成本偏差分 * 0.1

其中：
- 行业分：完全匹配 100，部分匹配 50，不匹配 0
- 规模分：Jaccard 相似度 * 100
- 模块分：Jaccard 相似度 * 100
- 成本偏差分：1 / (1 + |estimatedCost - targetCost| / targetCost) * 100
```

输出：`similarityScore` (0-100) + `matchBreakdown` (各维度得分详情)

#### 方案 B：分层阈值

| 分数区间 | 等级 | 前端展示 |
|---------|------|---------|
| 80-100 | 高度相似 | 绿色标签 "高度匹配" |
| 60-79 | 中等相似 | 黄色标签 "参考借鉴" |
| 40-59 | 低度相似 | 灰色标签 "轻度相关" |
| < 40 | 不相似 | 不展示 |

## 4. 差异化摘要视图

### 4.1 PreSales 视角
关注：客户复用、报价参考、工期估算

```json
{
  "projectName": "...",
  "customerName": "...",
  "industry": "...",
  "scale": "...",
  "similarityScore": 85,
  "estimatedDays": 120,
  "actualDays": 135,
  "dayVariance": "+12.5%",
  "estimatedCost": 500000,
  "actualCost": 520000,
  "costVariance": "+4%",
  "keyModules": ["库存管理", "生产管理"],
  "riskTags": ["多组织", "定制开发"]
}
```

### 4.2 PM 视角
关注：交付风险、团队配置、延期原因

```json
{
  "projectName": "...",
  "complexityLevel": "high",
  "teamSize": 8,
  "delayReason": "客户需求变更",
  "actualDays": 135,
  "estimatedDays": 120,
  "lessonsLearned": "...",
  "primaryRisks": ["scope creep", "integration delay"]
}
```

### 4.3 建议接口

```
GET /history/projects/:id/summary?view=presales|pm
```

返回对应角色的精简摘要，减少前端数据处理负担。

## 5. 优先级建议

| 优先级 | 事项 | 预期工作量 | 阻塞性 |
|--------|------|-----------|--------|
| P0 | 补 `projectName` 字段 | 小（schema + migration） | 阻塞列表展示 |
| P0 | 补 `customerName` 字段 | 小 | 阻塞老客户识别 |
| P1 | 相似度算法归一化 | 中（算法调整 + 测试） | 提升体验 |
| P1 | 新增摘要接口 `GET /summary` | 中 | 减少前端负担 |
| P2 | 补 `teamSize`, `complexityLevel` | 小 | 增强参考维度 |
| P2 | 分层阈值 + 前端标签 | 小 | 提升可读性 |
| P3 | `lessonsLearned`, `reusableAssets` | 小 | 长期知识沉淀 |
