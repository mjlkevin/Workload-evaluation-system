# 需求基线 V1（首发与迭代分层清单）

> 更新日期：2026-04-04  
> 评审口径：先判定是否阻塞首次上线（阻塞即首发），再按价值与依赖安排二轮、三轮。  
> 首发策略：严格 MVP。

## 1. 分层规则（本版冻结）

- `首发（Must）`：不完成会阻塞上线验收、合规或主链路演示。
- `二轮（Should）`：不阻塞首发，但对可运营性、可扩展性有明显增益。
- `三轮（Could）`：产品完整度与体验提升项，不影响首发与二轮主目标。

## 3. 首发冻结清单（Freeze List v1）

以下项定义为首发必达，不在本轮扩展到二轮/三轮需求：

1. 版本机制核心链路稳定回归（含权限、并发边界、错误语义）。
2. 四大评估流程可完整演示并可追溯到版本。
3. 认证与团队权限最小闭环可用。
4. 评审页最小闭环可演示。
5. 发布可复现（构建、测试、部署文档、环境模板）。
6. OpenAPI 与主线路由一致。

## 4. 不纳入首发范围（已明确）

- WBS 增强能力（编辑、持久化、运营能力）放入二轮。
- 集成与数据层改造放入二轮。
- 标准 PLM 完整策略与评审体验深度优化放入三轮。
- 数据库接入放入三轮，并以触发器作为前置条件。

## 5. 跟踪机制

- 需求状态字段统一：`待评估` -> `已分层` -> `开发中` -> `已验收`。
- 所有需求变更必须先更新本台账，再更新计划文档与任务看板。
- 若新增需求偏离当前计划，需先完成计划评审与冻结清单修订。
- 触发器治理：并发冲突、性能阈值、多实例扩容、审计追溯需求任一触发时，先更新计划后推进 DB。
