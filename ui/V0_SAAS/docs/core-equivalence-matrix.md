# 核心业务功能等价矩阵（四大模块）

本文档用于验收 `需求`、`实施评估`、`资源人天及成本`、`开发评估` 四个模块在新前端中的功能等价迁移进度。

## 1) 需求（Requirement Import）

| 功能点 | 旧前端行为 | 新前端目标 | 状态 |
| --- | --- | --- | --- |
| 版本选择与回读 | 支持选择需求版本并回填全部字段 | 支持版本下拉、选择后完整回填 | 待实现 |
| 基本信息编辑 | 客户/项目/商机/行业/营收/简介/目标等可编辑 | 保留核心字段可编辑并参与版本保存 | 待实现 |
| 业务需求条目编辑 | 可增删行，维护领域/分类/需求/提出人/定开标记 | 支持增删改、行级编辑 | 待实现 |
| 会议纪要录入 | 支持大文本录入 | 支持粘贴纪要并保存 | 待实现 |
| 保存草稿版本 | 点击保存后生成需求版本并持久化 | 真实接口持久化（`/api/v1/versions`） | 待实现 |
| 关联总方案版本 | 页面展示总方案版本并作为上下文 | 支持选择/展示并写入 payload | 待实现 |
| 错误提示 | 校验失败/接口失败有明确提示 | Toast/文案清晰提示 | 待实现 |

## 阻塞项与接口约束（当前）

1. 当前四模块页面多为“只读列表”，缺少版本读写闭环。
2. 导出链路尚未统一；MVP 采用前端 CSV 导出保证可用性。
3. WBS 接口在新前端有调用，但后端未稳定提供统一路由（需在契约收口阶段补齐）。
4. 版本为“不可变记录”，更新需“创建新版本”而非覆盖。

## MVP 验收口径（本轮执行）

1. 四模块全部支持：**编辑 -> 保存版本 -> 版本切换回读**。
2. 四模块全部支持核心计算（实施评估、资源成本、开发评估）。
3. 三个需要导出的模块（实施评估、资源人天、开发评估）提供可下载导出（CSV）。
4. 所有保存均走真实接口 `/api/v1/versions`，不依赖 mock 兜底。
