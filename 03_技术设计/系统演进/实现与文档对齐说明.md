# 实现与文档对齐说明（以代码为准）

> 本文档描述**当前仓库实现**与早期 V2 设计文档（轻量「仅计算导出」、无登录、三页面前端等）之间的差异，作为技术事实的单一补充来源。  
> V2 文档仍保留历史决策与评审结论；涉及「现状行为」请以本文 + `apps/api/src/main.ts`、`apps/web/src/App.vue` 为准。

## 1. 一句话摘要

系统在保留「模板 + 规则 + 计算 + 导出」核心的基础上，已演进为：**多页签业务工作台 + JWT 登录 + 文件型用户/推荐码/版本记录 + 浏览器端草稿（按用户隔离）+ 需求 Excel 智能解析（Kimi + 规则回退）**。

## 2. 技术栈（实现）

| 维度 | V2 文档常见表述 | 当前实现 |
|------|-----------------|----------|
| 后端框架 | NestJS 等 | **Express**（`apps/api/src/main.ts` 单入口聚合路由） |
| 前端 UI 库 | Element Plus | **无重型 UI 库**（`apps/web/package.json` 仅 `vue`） |
| 状态管理 | Pinia | **Vue Composition API**（`ref`/`computed`/`watch`，集中在 `App.vue`） |
| 鉴权 | 可选 / `X-Role` | **JWT `Authorization: Bearer`**；业务接口普遍要求登录；**不再信任客户端 `X-Role`** |
| 持久化 | 模板/规则/导出文件 | 另增 **`config/auth/users.json`、`config/auth/invite-codes.json`、`config/versions/records.json`** |

## 3. 后端 API 端点清单（实现）

Base：`http://localhost:3000`（`PORT` 可配）。API 前缀：`/api/v1`（下载见下表例外）。

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/v1/health` | 健康检查（无需登录） |
| POST | `/api/v1/auth/register` | 注册（需 body `inviteCode` 等） |
| POST | `/api/v1/auth/login` | 登录，返回 JWT |
| GET | `/api/v1/auth/me` | 当前用户（需 JWT） |
| POST | `/api/v1/auth/logout` | 登出（实现可为占位，以代码为准） |
| GET | `/api/v1/auth/users` | 用户列表（**admin**） |
| PATCH | `/api/v1/auth/users/:userId/status` | 启停用户（**admin**） |
| GET | `/api/v1/auth/invite-codes` | 推荐码列表（**admin**） |
| POST | `/api/v1/auth/invite-codes/generate` | 生成推荐码（**admin**） |
| GET | `/api/v1/versions` | 版本记录列表（已登录） |
| POST | `/api/v1/versions` | 创建版本记录（不可变覆盖策略以代码为准） |
| PATCH | `/api/v1/versions/:recordId/status` | 更新版本记录状态 |
| DELETE | `/api/v1/versions/:type/:versionCode` | 删除版本（**引用完整性**：被总方案引用则禁止） |
| GET | `/api/v1/templates` | 模板列表（需 JWT） |
| GET | `/api/v1/templates/:templateId` | 模板详情（需 JWT） |
| GET | `/api/v1/rule-sets/active` | 当前规则（需 JWT） |
| GET | `/api/v1/rule-sets/meta` | 规则元信息（需 JWT） |
| POST | `/api/v1/templates/import-json` | 导入模板 JSON（**admin**） |
| POST | `/api/v1/templates/import-excel` | 导入模板 Excel（**admin**） |
| POST | `/api/v1/rule-sets/import-json` | 导入规则 JSON（**admin**） |
| POST | `/api/v1/ai/parse-basic-info` | Excel 上传解析需求/基本情况等（Kimi + 规则回退，需 JWT） |
| POST | `/api/v1/estimates/calculate` | 计算（需 JWT） |
| POST | `/api/v1/estimates/calculate-and-export` | 计算并导出（需 JWT） |
| POST | `/api/v1/estimates/export/excel` | 导出 Excel（需 JWT） |
| POST | `/api/v1/estimates/export/pdf` | 导出 PDF（需 JWT） |
| GET | `/api/v1/exports/history` | 导出历史（需 JWT，**按 ownerUserId 隔离**） |
| POST | `/api/v1/sessions/start` | 会话开始（需 JWT） |
| POST | `/api/v1/sessions/:sessionId/calculate` | 会话计算（需 JWT） |
| GET | `/downloads/:fileName` | 下载导出文件（需 JWT，**校验文件归属**） |

> **注意**：下载路径为 **`/downloads/...`**，不在 `/api/v1` 下；与 V2 文档中「相对路径」描述一致，但调用时必须带鉴权头（前端多用 blob 下载）。

## 4. 角色模型（实现）

- 用户表字段角色为 **`admin` | `user`**（首注册用户为 `admin`）。
- 接口层将 **`user` 映射为与 V2「operator」等价的可估算角色**（`resolveApiRoleFromUser`）：普通用户可进行计算、导出、版本读写等业务；**仅 `admin` 可**模板/规则导入、用户与推荐码管理。

## 5. 版本与前端草稿（实现）

- **浏览器 `localStorage`**：实施评估、资源、总方案、需求导入、开发评估等草稿 **按 `userId` 分 key**；登出清理。
- **服务端 `config/versions/records.json`**：统一记录多类版本（实施/资源/需求导入/开发/总方案等），与前端保存联动；删除时校验是否被总方案引用。
- 版本前缀（产品约定，与界面一致）：`PG-`（实施）、`RS-`（资源）、`RI-`（需求导入）、`DV-`（开发）、`GL-`（总方案）。

## 6. 前端信息架构（实现）

单页应用内路由（`currentPage`）包括但不限于：

- **总览**：评估方案列表、新建方案向导、方案预览弹窗（关联关系 + 基本分析 + 子评估摘要）
- **需求导入**、**实施评估**、**开发评估**、**资源人天及成本**
- **WBS**、**评审**（占位）
- **用户管理**（admin）、**API**（文档展示）

与 V2「模板选择 / 录入 / 结果」三页静态拆分不一致；**三页逻辑已融合进「实施评估」工作台 + 结果区**。

## 7. 环境变量（实现摘录）

| 变量 | 用途 |
|------|------|
| `PORT` | API 端口，默认 `3000` |
| `JWT_SECRET` | JWT 签名密钥 |
| `JWT_EXPIRES_IN` | Token 有效期，默认 `8h` |
| `KIMI_API_KEY` | 需求 Excel 解析（Moonshot/Kimi） |
| `KIMI_MODEL` | 模型名，默认 `moonshot-v1-8k` |
| `KIMI_API_BASE_URL` | API 基址，默认官方地址 |

加载顺序：优先 `.env.local`，再 `.env`（见 `main.ts`）。

## 8. OpenAPI 维护说明

`docs/openapi.yaml` 仍为早期契约子集，且描述过 **`X-Role`**。当前运行时以 **JWT** 为准；完整路径以 §3 为准。后续可分批将 `openapi.yaml` 与 §3 对齐。
