# 团队协同（P0）权限矩阵

## 1. 角色映射（P0）

- `部门负责人` -> `manager`
- `实施顾问` -> `member`
- `售前顾问` -> `member`
- `销售顾问` -> `member`

> P0 采用 `manager/member` 两级权限执行，保留业务角色字段用于后续细化。

## 2. 资源与动作矩阵


| 资源           | 动作                 | manager | member | 说明          |
| ------------ | ------------------ | ------- | ------ | ----------- |
| Team         | 创建团队               | Y       | N      | 仅部门负责人      |
| Team         | 查看团队详情             | Y       | Y      | 仅限本人所属团队    |
| TeamMember   | 添加成员               | Y       | N      | 仅部门负责人      |
| TeamMember   | 移除成员               | Y       | N      | 仅部门负责人      |
| TeamMember   | 调整成员角色             | Y       | N      | 仅部门负责人      |
| TeamPlan(GL) | 查看团队方案列表           | Y       | Y      | 团队内只读可见     |
| TeamPlan(GL) | 变更方案归属团队           | Y       | N      | 手动关联策略      |
| Review       | 创建评审               | Y       | Y      | 团队成员可发起     |
| Review       | 查看评审详情             | Y       | Y      | 团队内可见       |
| Review       | 关闭评审（open->closed） | Y       | N      | 仅部门负责人      |
| Comment      | 发表评论               | Y       | Y      | 团队成员可评论     |
| Comment      | 查看评论列表             | Y       | Y      | 团队内可见       |
| AuditLog     | 查看留痕记录             | Y       | N      | P0 可先仅负责人可看 |


## 3. 服务端授权规则（最小实现）

每个团队协同接口按以下顺序校验：

1. JWT 鉴权通过（用户有效）。
2. 用户属于 `teamId` 对应团队。
3. 用户角色具备动作权限（基于矩阵）。
4. 对跨团队资源访问直接拒绝（`40301`）。

## 4. 审计留痕范围（P0）

需记录以下关键动作：

- 创建团队
- 添加/移除成员
- 调整成员角色
- 创建评审
- 关闭评审
- 发表评论

建议留痕字段：

- `actorUserId`
- `action`
- `targetType`
- `targetId`
- `teamId`
- `time`
