# R2 二轮执行 Backlog V1

> 目标：承接首发后工作，推进“集成与数据层能力 + WBS 增强”。
> 原则：未触发迁移条件前，不引入 DB 作为主路径；保持 repository 边界清晰。

## 周度拆分（建议）

### 第 1 周：数据层演进样板

- 任务 `R2-DATA-001`：确定迁移样板链路（建议 `versions` + `team`）。
  - 验收：输出链路图、读写边界与回滚策略文档。
- 任务 `R2-DATA-002`：补 repository 抽象一致性检查。
  - 验收：业务层不直接出现 JSON 字段路径依赖。
- 任务 `R2-DATA-003`：补 1 条并发冲突回归测试样例。
  - 验收：冲突场景可复现、错误语义稳定。

### 第 2 周：外部集成骨架

- 任务 `R2-INT-001`：定义外部集成契约（入参、出参、错误码）。
  - 验收：形成最小契约文档并评审通过。
- 任务 `R2-INT-002`：打通 1 条最小可验证链路（mock 或 sandbox）。
  - 验收：具备端到端调用日志与失败回退路径。
- 任务 `R2-INT-003`：补集成异常监控指标。
  - 验收：能统计成功率、失败原因、平均耗时。

### 第 3 周：WBS 增强

- 任务 `R2-WBS-001`：支持任务项结构化字段扩展（负责人、里程碑标记）。
  - 验收：接口返回结构稳定，前端可展示。
- 任务 `R2-WBS-002`：补任务状态表达与最小操作能力（非全量编辑）。
  - 验收：关键状态可维护，异常态反馈清晰。
- 任务 `R2-WBS-003`：完善 WBS 与版本追溯路径。
  - 验收：可从任务反查来源版本与记录。

### 第 4 周：收口与评审

- 任务 `R2-CLOSE-001`：联调回归与性能抽样。
  - 验收：核心链路回归通过，P95 未超过当前阈值。
- 任务 `R2-CLOSE-002`：更新文档与演示脚本。
  - 验收：新成员可按文档复现验证步骤。
- 任务 `R2-CLOSE-003`：形成 R3 输入包。
  - 验收：输出“遗留问题清单 + 三轮立项候选”。

## 迁移触发器观测（每周复盘）

- [ ] 并发写冲突 >= 每周 2 次人工修复。
- [ ] 核心查询 P95 超过阈值。
- [ ] 明确出现多实例/容器扩容需求。
- [ ] 审计、统计、权限追溯需求显著提升。

> 满足任一触发器时：先更新计划与风险评估，再进入 DB 立项流程。
