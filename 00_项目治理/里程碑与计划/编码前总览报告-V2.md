## 4.3 后续优化计划（新增）

- 新增“AI 友好调用”优化方向（Agent-Friendly API）：
  - 面向 Agent 提供更高层的评估接口，降低调用方参数负担
  - 将常见参数补全、合法性校验、错误修复建议下沉到后端
  - 支持基于用户自然语言问题返回可选模板/表格/字段与建议动作
  - 统一可解释响应格式，便于 Agent 向最终用户进行直观反馈
  - 建立 Agent 集成验收用例（多轮问答 -> 参数补全 -> 估算结果）

### 4.3.1 分阶段实施（P0/P1/P2）

1. P0（MVP，目标：2 周）
   - 交付高层接口：`POST /api/v1/agent/estimate`
   - 后端内置参数补全/校验/兜底
   - 统一可解释响应：`missingFields`、`assumptions`、`nextQuestions`
   - 最小会话态上下文支持（用于多轮补问）
   - 验收：Agent 在“缺失部分参数”情况下可完成估算并给出可读引导

2. P1（增强，目标：再 2~3 周）
   - 自然语言意图映射（用户问题 -> 模板/工作表/条目候选）
   - 候选确认与澄清机制（topK + 置信度 + 解释）
   - 导出链路 Agent 友好化（结构化说明 + 下一步动作）
   - 验收：复杂提问下补问轮次显著下降，成功率提升

3. P2（优化，目标：持续迭代）
   - 反馈闭环与规则优化
   - 调用质量看板（成功率、失败分类、平均回合数、补问次数）
   - Agent 契约版本治理（兼容/弃用/变更管理）
   - 验收：版本升级不破坏主要 Agent 调用场景

### 4.3.2 验收指标建议（可量化）

- 端到端成功率（Agent 发起 -> 产出估算结果）>= 95%
- 平均补问轮次 <= 2
- 参数校验类失败率较当前下降 >= 50%
- 接口字段变更兼容性：主流程零中断

### 4.3.3 P0 两周执行排期（W1/W2）

- W1（后端能力打底）
  - 交付 `POST /api/v1/agent/estimate`（单次估算 + 缺参提示）
  - 交付 `POST /api/v1/agent/session/start`（创建会话上下文）
  - 完成参数补全器（默认值、边界值修正、非法值解释）
  - 完成可解释响应模型（`status`、`missingFields`、`assumptions`、`nextQuestions`）
- W2（联调闭环）
  - 交付 `POST /api/v1/agent/session/:sessionId/continue`（多轮补问）
  - 提供 Agent 调试页（自然语言输入与结构化响应展示）
  - 完成 10+ 条场景回归（缺参、冲突、补问、导出）
  - 输出 Agent 使用示例（成功链路 + 失败修复链路）

### 4.3.4 P0 接口草案（面向 Agent）

1. `POST /api/v1/agent/estimate`
   - 入参（最小）：`userMessage`
   - 可选：`sessionId`、`hints`（如模板偏好、工作表偏好）
   - 出参核心：
     - `status`: `success | needs_clarification | failed`
     - `normalizedRequest`: 归一化后的估算请求（若已可计算）
     - `missingFields`: 仍缺失参数列表（若需补问）
     - `nextQuestions`: 建议追问列表（给 Agent 直接提问用户）
     - `assumptions`: 自动补全时采用的假设说明

2. `POST /api/v1/agent/session/start`
   - 入参：`userMessage`、可选 `hints`
   - 出参：`sessionId`、`contextSummary`、`nextQuestions`

3. `POST /api/v1/agent/session/:sessionId/continue`
   - 入参：`userMessage`
   - 出参：同 `agent/estimate`，用于多轮会话持续补全

### 4.3.5 P0 完成定义（DoD）
