# 前端视觉与交互规范 V2（Calendly 风格）

> **实现现状注记（2026-03）**：信息架构已从「固定三页」扩展为**多页签工作台**（总览、需求导入、实施/开发/资源评估等）；组件实现以 **Vue 3 + 自定义样式** 为主。下列布局与视觉 token 仍作为统一目标。

## 1. 定位

- 界面参考 **Calendly Analytics** 一类产品：**极简、干净、留白充足**。
- 本地参考图：`04_开发实现/前端/refero.design 7127269a-42c0-45e1-938f-0e0b2881a1ba.jpg`
- 与需求评审 **5.1~5.4** 对齐：三页流程、版本可见、先权威计算再导出、可解释结果与 `requestId`。

## 2. 布局骨架

- **左侧窄边栏**：Logo、主导航（模板选择 / 估算 / 结果与导出）、底部辅助入口（帮助、设置可选）。
- **主内容区**：顶栏标题 + 次级 Tab（若需要）、工具条（筛选/导出类操作）、下方 **卡片栅格**。
- **主操作按钮**：全宽或显著位置的 **品牌蓝实心按钮**（圆角 pill），次要操作为 **描边灰按钮**。

## 3. 色彩与表面


| 用途             | 建议                           |
| -------------- | ---------------------------- |
| 页面背景           | 浅灰 `#F5F5F5` ~ `#F2F2F2`     |
| 卡片/内容区         | 白 `#FFFFFF`，细浅灰描边或轻阴影        |
| 主色（主按钮、激活态、链接） | 品牌蓝 `#006BFF`（可按品牌微调，保持高对比）  |
| 主文             | 深灰/近黑                        |
| 次级说明           | 中灰小字                         |
| 分区标题           | 小号全大写 + 字间距（如 `CALCULATION`） |


## 4. 字体与层级

- 无衬线、现代感（如系统栈或 Inter / 苹方等）。
- **页面标题**：大号加粗。
- **卡片标题**：中号加粗；说明配小号信息图标 tooltip。
- **指标数字**：卡片内大号粗体（用于基准人天、总计等）。

## 5. 组件形态

- **卡片**：圆角约 `8px~12px`，内边距宽松；区块之间留白明显。
- **Tab**：下划线指示当前项（主色），非当前为灰字。
- **表格/列表**：行高舒适，斑马纹可选；分组行可用浅底区分。
- **空状态**：居中轻量文案（参考图中 “No data available” 气质）。
- **工具条**：日期/筛选类为 **描边下拉 + 图标**；右侧 `Customize` / `Export` 风格为浅灰描边按钮。

## 6. 三页映射（与 5.1 一致）

1. **模板选择页**：卡片列表或单列大卡片；展示 `templateVersion`、`ruleVersion`（只读徽章）。
2. **估算录入页**：左侧或顶部参数条（用户数、难度、多组织）；主体为可折叠分组 + 勾选；侧滑/抽屉展示 **规则摘要（只读）**。
3. **结果与导出页**：顶部 **指标卡**（基准、各增量、总计）；下方 **解释区**（`calculationBreakdown`）；底部 **导出前确认摘要**；提供 **复制 JSON**、`requestId` 复制。

## 7. 技术实现注意（Vue 3）

- 可使用 **Element Plus** 作基础组件，但必须 **主题覆盖**：主色、圆角、按钮样式、卡片、Tab 下划线，以贴近 Calendly，避免默认“后台风”。
- 建议集中维护：`design tokens`（CSS 变量或 SCSS）+ 少量全局样式覆盖。

## 8. 验收要点

- 首屏信息密度低、呼吸感强。
- 主路径（选模板 → 填参 → 计算 → 导出）一眼可辨。
- 移动端可后续迭代；V1 以桌面宽屏为主即可。
