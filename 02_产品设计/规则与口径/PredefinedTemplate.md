# 预置选择模式（实施评估）

## 1. 文档目的

说明「实施评估」工作区中 **预置选择模式** 的产品口径：用户一键命中一组模板条目（SKU 级 `templateItemId`），与手工勾选云产品/条目并存。

**实现位置**：`ui/V0_SAAS/app/dashboard/assessment/page.tsx`（预置模式交互）与 `ui/V0_SAAS/lib/workload-service.ts`（版本数据读写）。

## 2. 交互规则

- **互斥单选**：同一时间最多选中一种预置模式；切换模式会按新模式重算勾选集合（与模式相关的逻辑以代码为准）。
- **作用范围**：仅在 **「模块报价」** 工作表展示预置模式；切换到其他工作表时会清空已选预置项（见代码注释）。
- **与云产品关系**：选中预置后，系统根据配置中的 `templateItemIds` 勾选对应条目；用户仍可在预置基础上继续手工调整（含清除、自定义人天等，以当前页能力为准）。
- **未配置模式**：若某模式在当前模板中找不到对应条目，代码会尝试切换到该模式「首个可用工作表」——具体行为以当前 `assessment` 页面实现为准。

## 3. 模式清单与数据来源

| 模式名称 | 说明 | 样本来源（历史） |
|----------|------|------------------|
| 简单纯财 | 最小财务相关 SKU 组合 | 用户样本 `estimate-1774492409230.xlsx` |
| 标准财务供应链 | 在简单纯财基础上扩展供应链相关条目 | `estimate-1774493711566.xlsx` |
| 标准财务供应链生产 | 在标准财务供应链基础上增加制造相关条目 | `estimate-1774494435787.xlsx`（标准财务供应链制造） |
| 研产供销一体化 | 更广的研产供销条目集合 | `estimate-1774537946495.xlsx` |
| 标准PLM | **占位**：当前 `templateItemIds` 为空数组，选中后无默认勾选，待产品补全条目清单 | 待定义 |

## 4. 配置结构（维护约定）

每种模式在代码中的结构为：

```ts
Record<模式名, { templateItemIds: string[] }>
```

- `templateItemIds` 对应当前加载模板中的 **条目 ID**（非业务名称）。**模板重新导入或 ID 变化后，须同步更新本配置**，否则预置会勾选错误或静默跳过。

## 5. 与文档/规则的关系

- 预置模式只解决 **「默认勾选哪些模板项」**，不改变规则引擎中的用户数分段、难度、多组织等计算口径。
- 正式口径仍以 `config/rules/*.json` 与 `03_技术设计` 下规则文档为准。

## 6. 变更记录建议

在调整 `presetModeConfigs` 的合并提交中，建议在提交说明或本文件末尾追加一行：**日期 + 变更模式 + 关联 Excel/需求单**，便于审计。
