需求确认梳理 — 最终版
版本号规则（核心规则）
格式：{单据基础编号}-V{字母}{自然数}

创建首次检入       → XXX-VA1
再次检出 → 检入    → XXX-VA2
再次检出 → 检入    → XXX-VA3
升版               → XXX-VB1（字母+1，数字重置为1）
再次检出 → 检入    → XXX-VB2
再次升版           → XXX-VC1
...
字母溢出（Z后）    → 需确认，建议用 AA、AB...
完整状态机
[新建单据]
  └─→ 未检入状态（无版本号，草稿）
        │
        │ 显式点击【检出】按钮（写入锁 + 操作人）
        ▼
      已检出 + 修订中（其他用户见锁，不可检出）
        │  ↑
        │  │ 编辑 / 保存（不改变状态）
        │  └──────────────────────
        │ 显式点击【检入】
        ▼
      已检入 + 修订中  →  版本号更新为 XXX-VA1/VA2/...
        │
        │ 【升版】操作（只在已检入可触发）
        │   ├─ 当前版本归档（历史快照，只读）
        │   └─ 新版本创建
        │         └─→ 已检出 + 修订中（XXX-VB1，字母+1，数字归1）
        │               │ 用户检入
        │               └─→ 已检入 + 修订中（VB1 确认）
        │
        │ 【总方案评审通过】（外部触发，级联）
        ▼
      已检入 + 已审核  ←── 整体只读，禁止检出
        │
        │ 如需修改 → 必须先【升版】
        └─→ 新版：已检出 + 修订中（XXX-VB1）

        字段,类型,取值,说明
检出状态,enum,已检出 / 已检入,排他锁标记
检出人,string,用户ID,检出时写入，检入时清空
版本状态,enum,修订中 / 已审核,审核由总方案评审级联触发
版本号,string,XXX-VA1 格式,每次检入递增数字，升版递增字母
创建时间,datetime,—,单据首次创建时间
修改时间,datetime,—,每次保存更新
审核时间,datetime,—,评审通过时写入

补充确认的问题
1. 撤销检出（放弃修改） 用户检出后，发现不需要改动，是否支持撤销检出（释放锁，丢弃修改，恢复到上次检入状态）？这是工程实践中的高频需求，建议支持。

2. 升版的触发条件 当前确认：已检入（修订中 or 已审核都可升版）。但建议：

修订中 + 已检入 → 允许升版（常规迭代）
已审核 + 已检入 → 也允许升版（审核后继续迭代）
是否两种情况都允许？还是只有 已审核 后才能升版？

3. 检出锁超时机制 排他锁场景下，如果用户 A 检出后长时间未操作（离职/忘记检入），是否需要管理员强制解锁功能？

4. 升版后老版本的版本状态 老版本归档时，版本状态保留原值还是统一改为已归档（建议新增一个归档状态以便区分）？

5. 新建单据的版本号起点 单据刚创建、尚未首次检入前，版本号显示什么？建议选项：

显示 草稿（无编号）
显示 XXX-VA0（预分配）
确认以上 5 点后即可开始实现
