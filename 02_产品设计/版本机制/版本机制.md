# 版本检入检出升版机制

实现版本管理的检出（排他锁）、检入、撤销检出、升版、管理员解锁全套机制，覆盖五个模块：需求(RI)、实施评估(PG)、资源人天(RS)、开发评估(DV)、总方案(GL)。

---

## 版本号规则

```
原始单据编号（基础码）：{PREFIX}-{YYYYMMDD}{SEQ}{LETTER}
             例如：RI-20260404001A

带检入轮次的版本号：{基础码}-V{大写字母}{自然数}
             例：RI-20260404001A-VA1（首次检入）
                 RI-20260404001A-VA2（再次检出→检入）
                 RI-20260404001A-VB1（升版后首次检入）
```

- 每次**检入**：尾部自然数 +1（A1→A2→A3）
- 每次**升版**：字母 +1，自然数归 1（A3→B1）
- 字母溢出 Z 后使用 AA、AB…（本期仅实现 A-Z，留扩展）
- 新建单据未检入前：显示"草稿"，无版本号后缀

---

## 新增字段（VersionRecord）

字段类型说明`checkoutStatus"checked_in" | "checked_out"`检出状态`versionStatus"drafting" | "reviewed"`版本状态`checkedOutByUserIdstring?`检出人 ID`checkedOutByUsernamestring?`检出人名`checkoutAtstring?`检出时间`majorLetterstring`升版字母（A/B/C…）`minorNumbernumber`检入轮次（1/2/3…）`baseCodestring`单据基础码（不含 -Vxx 后缀）`isHistoricalArchiveboolean`是否历史归档（升版后旧版本）`archivedAtstring?`归档时间

---

## 新增 API 端点

方法路径说明POST`/api/v1/versions/:id/checkout`显式检出（写入排他锁）POST`/api/v1/versions/:id/checkin`检入（释放锁，版本号自然数+1）POST`/api/v1/versions/:id/undo-checkout`撤销检出（丢弃修改，释放锁）POST`/api/v1/versions/:id/promote`升版（归档当前，创建新版本 VX1）PATCH`/api/v1/versions/:id/force-unlock`管理员强制解锁

---

## 前端操作按钮

每个模块版本区域新增操作栏：

```
[检出]  [检入]  [撤销检出]  [升版]   状态标签: 已检入·修订中
```

- **检出**：仅在 `checked_in + drafting + 非历史归档` 时可点
- **检入**：仅在 `checked_out + 当前用户` 时可点
- **撤销检出**：仅在 `checked_out + 当前用户` 时可点
- **升版**：仅在 `checked_in + drafting` 时可点（已审核也允许）
- **管理员解锁**：仅 admin 可见，在 `checked_out` 时显示
